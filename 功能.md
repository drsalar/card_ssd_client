## 主页功能

### 0. 背景音乐

- 进入画面后按本地缓存的开关状态自动播放 `audio/bgm2.mp3`。
- 首页连接状态下方提供 `♫ 开/关` 控制，切换后会写入本地缓存；按钮绿色表示开启、红色表示关闭。

### 1. 创建房间

- **生成牌桌 ID**：创建 4 位数字的牌桌 ID。
- **选择游戏规则**：规则选择完成后，进入对局页面。
  - **是否带马牌**
  - **对局局数**：支持 `5`、`10`、`15`、`20` 局。

### 2. 加入房间

- 打开弹窗，输入牌桌 ID。
- 校验通过后，进入对局页面。

## 对局页面功能

### 1. 牌桌布局

- 按玩家人数围成一圈展示牌桌布局。
- 牌桌为横向拉长的自适应椭圆，打牌阶段会适当增加横纵尺寸，给三道放置区留出更完整的桌面空间。
- Canvas 按设备像素比渲染，真机高分屏下文字与线条保持清晰。
- 房间顶部信息、退出按钮和调试按钮会避开刘海/状态栏/微信胶囊区域。
- 对局页连接状态下方同样提供 `♫ 开/关` 控制，与首页共用同一缓存状态；按钮颜色同样用于区分当前开关状态。
- 支持 `2-6` 人参与游戏。
- 每位玩家使用微信头像和昵称作为标识，昵称、积分、准备/出牌中/已开牌/BOT 徽标在手机分辨率下加粗放大展示。
- 玩家头像旁显示当前积分。

### 2. 准备阶段

- 玩家可以点击准备，也可以随时取消准备。
- 当所有玩家都准备好后，开始对局。

### 3. 发牌与手牌区

- 每人随机发放 13 张牌，并放入手牌区。
- 手牌区会较底部留出更大间距，避免贴底；放置区围绕牌桌中心展示，并与底部自己头像和手牌操作区保持安全间距。
- 手牌支持点击单张牌或滑动经过多张牌来切换选中状态。
- 手牌下方提供牌型快捷按钮：`五龙`（仅 5 人以上显示）、`同花顺`、`炸弹`、`葫芦`、`同花`、`顺子`、`三条`、`对子`。
- 当前手牌可组成对应牌型时按钮高亮可点击，否则灰色禁用；重复点击同一高亮按钮会轮换选中不同组合。
- 手牌支持排序方式：
  - 按花色排序
  - 按点数排序
- 加色牌沿用同花色的红黑颜色显示，避免不同区域的牌面颜色不一致。
- 画面中间显示头道、中道、尾道的空白放置区。
- 每一道放置区右侧都有取消按钮，点击后可将该道牌返回手牌中。
- 可点击“放入”按钮或直接点击放置区，将当前选中的手牌放入对应道。

### 4. 放牌规则

- 玩家收到牌后，可以选择 3 张或 5 张牌，放入对应道中。
- 放入放置区后的道牌会按点数从大到小排序展示。
- 放置区牌型提示显示在左侧标签栏底部，避免与道牌和操作按钮重叠。
- 若选择牌数不符合对应道要求，则提示不允许放入。
- 放入 2 道后，剩余牌默认放入第 3 道。
- 三道牌必须按规则从小到大放入。
- 道牌校验结果通过放置区背景色提示：
  - **绿色**：该道正确。
  - **红色**：该道错误。
- 三道均正确放入后，可以选择开牌。
- 开牌后不允许再改动。

### 5. 比牌流程

所有玩家都开牌后，进入比牌流程：

1. 隐藏放置区。
2. 所有玩家依次按 **头道 → 中道 → 尾道** 的顺序比较。
3. 每次比较时，在各自头像合适的位置显示：
   - 对应道信息（牌面会随人数缩放，并按座位自动靠左/靠右/上方放置；`3-6` 人时左右两侧含底部左右座位统一形成左侧上移、右侧下移的固定高度差，避免对比区域重叠并方便区分）
   - 两两比较结果
   - 当前对局的积分增减（+X 字号 16px，牌型名 13px；比较阶段会隐藏头像旁常驻总积分和开牌徽标，避免多个信息重叠）
4. 当前道展示完成后，进入下一道比较。
5. 三道比较完成后，如有打枪则显示打枪信息和积分增减，并播放从胜利方飞向失败方的枪击弹道动画，同时播放 `audio/bullet.mp3`；没有打枪则跳过该阶段。
6. 打枪信息显示完成后，如有全垒打则显示全垒打信息和积分增减，同时播放 `audio/boom.mp3`；没有全垒打则跳过该阶段。
7. 所有对比完成后，刷新对局积分信息。
8. 在牌桌上弹出本局统计信息，等待玩家确认。

### 6. 本局统计信息

统计信息包含：

- 每位玩家的出牌信息。
- 每位玩家的积分。
- 按道显示的本道得分（已合并冲三 / 中道牌型 / 尾道牌型等特殊加分，及打枪 ×2、本垒打 ×2、马牌 ×2 等全部倍率），三道相加严格等于该玩家本局总分。
- 玩家行展示打枪目标、全垒打、马牌标记，便于核对谁打枪了谁。
- **本垒打判定支持 2-6 人**：玩家三道全胜对局中所有对手时即触发本垒打，在打枪 ×2 基础上再 ×2 加倍。2 人场只要打枪对方一人即同时构成本垒打。
- 结算面板针对 `2-6` 人自适应行高；人数较多时支持上下滑动查看完整列表。
- 结算面板整体采用紧凑布局，以 `5` 人场无需滚动即可完整显示为标准。
- 每道牌使用缩小后的扑克牌左上角裁剪图按点数从大到小展示点数与花色，并在右侧分列显示牌型和该道分数。
- 符合特殊加分的牌型会在结算面板中追加加分标记并变色：额外 `1-2` 分为蓝色、`3-4` 分为橙色、`5-10` 分为红色（加分标记仅作牌型说明，对应分值已计入右侧本道得分，不会重复计入）。
- 持有马牌（红桃 5）的玩家，每一道每一项积分均在原有基础上额外 `×2`，与打枪、本垒打的倍率累乘生效。

### 7. 下一局

- 所有玩家点击确认后，关闭统计信息。
- 玩家点击准备按钮，准备进行下一局。

### 8. 对局结束

- 所有对局完成后，显示总积分排行。

## 主页 HTTP 化 / 对局 WebSocket / 头像离线视觉

为降低大厅闲置连接成本、并让玩家一眼看出队友 / 对手的网络状态，网络分层重新划分为：

### 1. 大厅仅使用 HTTP

- 客户端启动后不再主动连接 WebSocket，进入大厅后仅调用一次 `POST /api/login`（载荷 `{ code, nickname, avatarUrl }`，其中 `code` 由 `wx.login` 异步获得）拿到 `activeRoom` 摘要；返回 `null` 则隐藏“重新进入”按钮。服务端会以 `code` 调 `https://api.weixin.qq.com/sns/jscode2session` 解出真实 openid 随响应返回（失败时退化读请求头 `X-WX-OPENID`），客户端拿到 `res.openid` 后写回本地存储与 `databus.user.openid`。
- 如需主动刷新可调用 `GET /api/lobby/active-room?openid=xxx`；两个接口均为只读，不会修改服务端任何在线状态。
- 登录失败时顶部 Toast 提示“登录失败，点击重试”，点击屏幕即可重发请求。

### 2. 仅在入房动作时升级 WebSocket

- “创建房间 / 加入房间 / 重新进入”三个入口会先 `socket.connect()`，等待 `LOGIN_OK` 后再发送 `CREATE_ROOM` / `JOIN_ROOM`。如 Socket 已连接则复用。
- 在“返回大厅”场景（收到 `LEAVE_ROOM_OK`、或 `MATCH_END` 后点击“返回大厅”）会主动 `socket.close()` 断开连接。

### 2.1 云托管通道（默认）

- 所有 HTTP / WebSocket 默认走微信云托管：`wx.cloud.callContainer` 与 `wx.cloud.connectContainer`，无需在小游戏后台配置 request/socket 合法域名。
- 云环境通过全局变量集中配置：`GameGlobal.CLOUD_ENV`（默认 `prod-d1gy3h2lh5a169861`）、`GameGlobal.CLOUD_SERVICE`（默认 `golang-8gye`），可在外部预先注入覆盖。
- 非小游戏环境（浏览器调试）或缺失 `wx.cloud` 时，自动降级到 `wx.request` / `fetch` + `wx.connectSocket` / `WebSocket`，此时使用 `GameGlobal.HTTP_BASE` / `GameGlobal.SOCKET_URL` 作为兜底地址。

### 3. 仅对局中执行自动重连

- `socket_client` 仅在 `databus.scene === SCENES.ROOM` 时才进行 5 次 / 1.5 秒间隔的自动重连；大厅场景下任何异常关闭都不会触发重连。
- 重连后发送 `LOGIN`、在 `LOGIN_OK` 后自动 `JOIN_ROOM`；服务端返回 `ROOM_NOT_FOUND` 时清空本地 `databus.room`、切回大厅并提示“房间已结束”。

### 4. 微信小游戏生命周期对接

- `wx.onShow`：若处于对局且 socket 已断，立即触发重连；实现“切后台 30 秒以内”返回即无缝继续对局。
- `wx.onHide` 不主动断开 Socket，依赖原生层保活。

### 5. 头像离线视觉

- 服务端在玩家 `Offline=true` 状态变更时立即广播 `ROOM_STATE`；玩家重连后同样立即广播一次。
- 客户端在受影响玩家（非“我”、非 Bot）的头像上叠加 `rgba(0,0,0,0.45)` 圆形蒙层，叠加白色“OFF”字样徽章（约头像直径 60%）。
- 玩家从离线恢复（`offline=false`）后蒙层与徽章自动移除；本地玩家本人不渲染蒙层（以顶部“重连中…” Toast 提示为准），Bot 玩家从不渲染蒙层。

## 断线重连

针对网络抖动、切后台过久或小游戏被强制关闭等场景，提供「重连」/「重新进入」能力：

### 1. WebSocket 自动重连

- 客户端检测到连接断开后，自动尝试重新连接最多 5 次，间隔 1.5 秒。
- 重连过程中顶部 Toast 提示「重连中…(n/5)」。
- **握手超时兜底**：云托管首次冷启动时 `connectContainer` 偶发既不 open 也不报错的情况，客户端会在 8 秒后强制视为失败并触发重连，避免「连接中…」状态永久卡死。
- **大厅强制重连**：用户在大厅点击「创建/加入房间/重新进入」时，若发现 socket 仍卡在 `connecting` 状态，会强制销毁旧 socket 并立即重新发起握手。

### 2. 业务层自动恢复

- 重连成功后自动重发 `LOGIN`；若本地仍持有 `databus.room`，则在收到 `LOGIN_OK` 后自动发送 `JOIN_ROOM` 回原房间。
- 服务端命中重连分支时，会先单播一条 `RECONNECT_SNAPSHOT`：携带当前阶段、手牌、已提交的三道、最近一局的结算结果、对局进度。
- 客户端据此还原本地数据，并按阶段进入对应子界面（摆牌 / 等待开牌 / 比牌 / 整场结束）。

### 3. 大厅「重新进入」入口

- 登录成功后，若服务端检测到该 openid 仍在某个未结束房间，会在 `LOGIN_OK.data.activeRoom` 中带回房间信息。
- 大厅页主按钮上方显示「重新进入（房间号）」按钮；点击后直接回到该房间，恢复全部对局状态。
- 若房间已结束，按钮自动隐藏并提示「房间已不存在」。

### 4. 服务端 30 秒兜底

- 玩家断线后阶段化兜底（30 秒为窗口）：
  - **准备阶段**：连接断开瞬间即移除该玩家
  - **对局阶段（playing）**：标记 `offline=true`，30 秒内回来视为重连；30 秒后未回则按散牌（头 3/中 5/尾 5）参与本局结算
  - **比牌阶段（comparing）**：保留座位（结算结果与积分已固定），等待重连或整场结束
  - **整场结束（match_end）**：30 秒后未回则移除该玩家，避免永久占座
- **进行中房间 24 小时保活**：当对局已开始的房间内所有真人玩家都离线时，服务端不再立即销毁，而是登记 `AllOfflineSince` 时间戳；任意一名真人在 24 小时内重连即可恢复全部对局状态。每小时一次的全局巡检会销毁“持续 24 小时无任何真人在线”的房间。
- 同 openid 新建连接时，服务端会断开旧连接、把旧 Session 的房间归属继承到新 Session，并取消该玩家的弃局兜底计时器。

### 5. 返回主页 / 投票解散 / 最新房间优先

- **返回按钮**：房间页右上角原“退出”按钮文案改为“返回”。点击后只断开 WebSocket 并切回大厅，不发送 `LEAVE_ROOM`；房间会保留 24 小时供稍后从主页“重新进入”继续对局。服务端进程重启后会从 MySQL 加载未销毁房间还原到内存（参见 `card_ssd/README.md` 「MySQL 持久化」章节）。仅整场结束阶段才会真正发送 `LEAVE_ROOM` 离开。
- **投票解散**：在 `playing/comparing` 阶段，真人玩家可以点击右上角按钮组中的“解散”按钮发起 `VOTE_DISSOLVE`，“解散”与“返回”按钮并排显示并避开牌桌。所有在线真人都同意时立即按当前积分提前进入 `match_end` 排行；电脑玩家默认视为同意。`playing` 阶段触发会跳过本局未发出的结算；`comparing` 阶段触发则保留本局已发结算结果。投票从首次发起开始 60 秒未达成一致即作废，所有投票被清空并广播 `VOTE_DISSOLVE_TIMEOUT`。
- **最新房间优先**：当一个 openid 同时挂在多个未结束房间时，登录响应只返回 `LastActiveAt` 最新的一条 `activeRoom`，避免“重新进入”指向旧局。

## 微信头像 / 昵称展示

主页与对局中本地玩家及对手的头像、昵称统一使用微信官方资料展示，让玩家一眼辨认彼此身份。

### 1. 客户端获取微信资料

- 微信小游戏自基础库 2.27.1 起，`wx.getUserInfo` 返回的昵称被强制改为「微信用户」、头像为灰色默认；`wx.getUserProfile` 在小游戏端已废弃；`wx.createUserInfoButton` 的回调同样拿不到真实资料。本项目在大厅页采用官方推荐的「头像昵称填写能力」：
  - **头像**：`wx.chooseMedia({ mediaType: ['image'] })` 让玩家从相册/相机选一张图片，使用 `tempFilePath` 作为头像 URL；
  - **昵称**：`wx.showModal({ editable: true, placeholderText: '请输入昵称（最多 12 字）' })` 让玩家手动输入昵称。
- **主页随时改名/换头像**：点击主页顶部「头像+昵称」区域会弹 ActionSheet，提供`[换头像 / 改昵称]`两个选项。
- 启动时 [main.js](./js/main.js) `initUser()` 仅读本地缓存 `wx_user_info` 填充 `databus.user.nickname/avatarUrl`，上次填过资料的玩家下次进入同样看到自己的头像昵称，无需重新填写。
- 未填写时兜底使用「玩家 + openid 后4位」并在主页头像下提示「点击头像设置头像与昵称」，不阻塞主流程。任一环节取消均静默，不写缓存。

### 2. 主页展示

- [lobby_scene.js](./js/scenes/lobby_scene.js) 顶部在玩家昵称左侧增绘 36px 圆形头像（复用 [Avatar](./js/ui/Avatar.js) 的图片加载与首字符兜底）。
- `onTouchStart` 命中「头像+昵称」区域时调 `wx.showActionSheet({itemList:['换头像','改昵称']})`，再分别用特定 mode 调用 [pickUserInfo](./js/utils/wx_user.js)（`avatar`只选图，`nickname`只填名，`both`串行）。
- 两项任一成功即写回 `databus.user` 与 storage，调 `selfAvatar.setUrl(user.avatarUrl)` 重绘，并 `eventBus.emit('user_info_updated')` 触发重新调 `POST /api/login` 同步服务端 session。

### 3. 对局展示

- 房间页 [room_scene.js](./js/scenes/room_scene.js) 调用 [PlayerSeat](./js/ui/PlayerSeat.js) 渲染所有玩家座位，头像 URL 直接来自 `ROOM_STATE.players[*].avatarUrl`，加载失败回退为昵称首字符；URL 变化时 `Avatar.setUrl` 自动重新加载。
- 结算面板 [settle_phase.js](./js/scenes/settle_phase.js) 在每行玩家行的左侧额外增绘 22px 行内头像，复用 `PlayerSeat.getAvatar(player)` 缓存避免重复加载。
- 电脑玩家 (`isBot=true`) 保留「电脑X」昵称与无头像 + BOT 角标的现有兜底视觉。

### 4. 字段流向

```
主页头像点击 ActionSheet → pickUserInfo (chooseMedia / showModal) → { nickname, avatarUrl }
              → databus.user (nickname/avatarUrl) + storage 缓存
              → POST /api/login 上报 → Session.Nickname/AvatarUrl
              → WS LOGIN 帧上报（_autoLogin 从 databus 实时读取）
              → Player.Nickname/AvatarUrl（进房 / Upsert 同步）
              → ROOM_STATE.players[*].nickname/avatarUrl 广播到同房间
```

## 调试日志面板

为方便开发者在真机或微信开发者工具外定位卡顿、协议问题，前端新增「调试日志面板」开发期模块（默认开启，发布时一行关闭）。

### 1. 入口

- 画布右上角浮动 `LOG` 按钮默认隐藏，避免遮挡游戏界面；需要临时查看时，可在 [main.js](./js/main.js) 顶部将 `GameGlobal.DEBUG_LOG_BUTTON` 置为 `true`；
- 按钮开启后会避让微信右上胶囊菜单；房间页位于「退出」按钮左侧 8px，避免重叠；
- 当面板关闭、缓冲区中存在新增 `ERROR` 日志时，按钮右上角显示红点提示；点击面板后清零。

### 2. 日志收集

- **Console**：替换 `console.log/info/warn/error/debug`，原始输出仍保留至开发者工具 Console；
- **HTTP**：所有 `wx.request` / `fetch` 请求记录方法、URL、请求体、状态码、耗时与响应体（自动屏蔽 `token` 字段）；
- **WebSocket**：连接事件 `connect/open/close/error/reconnect`，以及每条收发消息（type、reqId、payload 摘要）；`ERROR` 协议升级为 ERROR 级别、`RECONNECT_SNAPSHOT` 升级为 WARN 级别。
- 缓冲区上限默认 500 条，超限按头部丢弃。

### 3. 面板交互

- 顶部分类标签：`ALL / CONSOLE / HTTP / WS / ERROR`，单击即时过滤；
- 列表区按 `HH:mm:ss.sss [LEVEL] [SOURCE] 内容` 形式滚动展示，按级别/来源着色，超长内容截断；
- 上下滑动查看历史；松手后若处于贴底位置则继续自动跟随新日志，否则停留；
- 工具栏：
  - **暂停/恢复**：暂停期间缓冲区继续写入，但视图不再贴底刷新；
  - **清空**：清空全部缓冲；
  - **发包**：通过两步 `wx.showModal` 输入 `type` 与 JSON `data`，校验通过后调用 `socket.send(type, data)`；JSON 格式错误或 Socket 未连接时给出 Toast 提示并阻止发送；
  - **会话**：弹出当前 `databus.user / scene / room / socket` 关键字段快照，可一键复制；
- 单击单条日志弹出详情视图（不截断），提供「复制」按钮调用 `wx.setClipboardData` 写入剪贴板。

### 4. 关闭方式

在 [main.js](./js/main.js) 顶部将 `GameGlobal.DEBUG_LOG` 置为 `false`，即可一行关闭按钮、面板与所有 hook（不再拦截 `console`、不再写入网络日志），用于发布版本零开销。
