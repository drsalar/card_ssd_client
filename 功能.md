## 主页功能

### 1. 创建房间

- **生成牌桌 ID**：创建 4 位数字的牌桌 ID。
- **选择游戏规则**：规则选择完成后，进入对局页面。
  - **是否带马牌**
  - **对局局数**：支持 `5`、`10`、`15`、`20` 局。

### 2. 加入房间

- 打开弹窗，输入牌桌 ID。
- 校验通过后，进入对局页面。

## 对局页面功能

### 1. 牌桌布局

- 按玩家人数围成一圈展示牌桌布局。
- 牌桌为横向拉长的自适应椭圆，会根据屏幕高度调整纵向半径。
- 支持 `2-6` 人参与游戏。
- 每位玩家使用微信头像和昵称作为标识。
- 玩家头像旁显示当前积分。

### 2. 准备阶段

- 玩家可以点击准备，也可以随时取消准备。
- 当所有玩家都准备好后，开始对局。

### 3. 发牌与手牌区

- 每人随机发放 13 张牌，并放入手牌区。
- 手牌区会较底部留出更大间距，避免贴底；放置区下移并靠近手牌区，方便对比三道与手牌。
- 手牌支持点击单张牌或滑动经过多张牌来切换选中状态。
- 手牌支持排序方式：
  - 按花色排序
  - 按点数排序
- 加色牌沿用同花色的红黑颜色显示，避免不同区域的牌面颜色不一致。
- 画面中间显示头道、中道、尾道的空白放置区。
- 每一道放置区右侧都有取消按钮，点击后可将该道牌返回手牌中。
- 可点击“放入”按钮或直接点击放置区，将当前选中的手牌放入对应道。

### 4. 放牌规则

- 玩家收到牌后，可以选择 3 张或 5 张牌，放入对应道中。
- 放入放置区后的道牌会按点数从大到小排序展示。
- 若选择牌数不符合对应道要求，则提示不允许放入。
- 放入 2 道后，剩余牌默认放入第 3 道。
- 三道牌必须按规则从小到大放入。
- 道牌校验结果通过放置区背景色提示：
  - **绿色**：该道正确。
  - **红色**：该道错误。
- 三道均正确放入后，可以选择开牌。
- 开牌后不允许再改动。

### 5. 比牌流程

所有玩家都开牌后，进入比牌流程：

1. 隐藏放置区。
2. 所有玩家依次按 **头道 → 中道 → 尾道** 的顺序比较。
3. 每次比较时，在各自头像合适的位置显示：
   - 对应道信息
   - 两两比较结果
   - 当前对局的积分增减
4. 当前道展示完成后，进入下一道比较。
5. 三道比较完成后，如有打枪则显示打枪信息和积分增减，并播放从胜利方飞向失败方的枪击弹道动画；没有打枪则跳过该阶段。
6. 打枪信息显示完成后，如有全垒打则显示全垒打信息和积分增减；没有全垒打则跳过该阶段。
7. 所有对比完成后，刷新对局积分信息。
8. 在牌桌上弹出本局统计信息，等待玩家确认。

### 6. 本局统计信息

统计信息包含：

- 每位玩家的出牌信息。
- 每位玩家的积分。
- 按道显示的基础积分。
- 玩家行展示打枪次数、全垒打、马牌标记。
- 每道牌使用扑克牌左上角裁剪图按点数从大到小展示点数与花色，并在右侧显示牌型和该道分数。

### 7. 下一局

- 所有玩家点击确认后，关闭统计信息。
- 玩家点击准备按钮，准备进行下一局。

### 8. 对局结束

- 所有对局完成后，显示总积分排行。

## 主页 HTTP 化 / 对局 WebSocket / 头像离线视觉

为降低大厅闲置连接成本、并让玩家一眼看出队友 / 对手的网络状态，网络分层重新划分为：

### 1. 大厅仅使用 HTTP

- 客户端启动后不再主动连接 WebSocket，进入大厅后仅调用一次 `POST /api/login`（载荷 `{ openid, nickname, avatarUrl }`）拿到 `activeRoom` 摘要；返回 `null` 则隐藏“重新进入”按钮。
- 如需主动刷新可调用 `GET /api/lobby/active-room?openid=xxx`；两个接口均为只读，不会修改服务端任何在线状态。
- 登录失败时顶部 Toast 提示“登录失败，点击重试”，点击屏幕即可重发请求。

### 2. 仅在入房动作时升级 WebSocket

- “创建房间 / 加入房间 / 重新进入”三个入口会先 `socket.connect()`，等待 `LOGIN_OK` 后再发送 `CREATE_ROOM` / `JOIN_ROOM`。如 Socket 已连接则复用。
- 在“返回大厅”场景（收到 `LEAVE_ROOM_OK`、或 `MATCH_END` 后点击“返回大厅”）会主动 `socket.close()` 断开连接。

### 2.1 云托管通道（默认）

- 所有 HTTP / WebSocket 默认走微信云托管：`wx.cloud.callContainer` 与 `wx.cloud.connectContainer`，无需在小游戏后台配置 request/socket 合法域名。
- 云环境通过全局变量集中配置：`GameGlobal.CLOUD_ENV`（默认 `prod-d1gy3h2lh5a169861`）、`GameGlobal.CLOUD_SERVICE`（默认 `golang-8gye`），可在外部预先注入覆盖。
- 非小游戏环境（浏览器调试）或缺失 `wx.cloud` 时，自动降级到 `wx.request` / `fetch` + `wx.connectSocket` / `WebSocket`，此时使用 `GameGlobal.HTTP_BASE` / `GameGlobal.SOCKET_URL` 作为兜底地址。

### 3. 仅对局中执行自动重连

- `socket_client` 仅在 `databus.scene === SCENES.ROOM` 时才进行 5 次 / 1.5 秒间隔的自动重连；大厅场景下任何异常关闭都不会触发重连。
- 重连后发送 `LOGIN`、在 `LOGIN_OK` 后自动 `JOIN_ROOM`；服务端返回 `ROOM_NOT_FOUND` 时清空本地 `databus.room`、切回大厅并提示“房间已结束”。

### 4. 微信小游戏生命周期对接

- `wx.onShow`：若处于对局且 socket 已断，立即触发重连；实现“切后台 30 秒以内”返回即无缝继续对局。
- `wx.onHide` 不主动断开 Socket，依赖原生层保活。

### 5. 头像离线视觉

- 服务端在玩家 `Offline=true` 状态变更时立即广播 `ROOM_STATE`；玩家重连后同样立即广播一次。
- 客户端在受影响玩家（非“我”、非 Bot）的头像上叠加 `rgba(0,0,0,0.45)` 圆形蒙层，叠加白色“OFF”字样徽章（约头像直径 60%）。
- 玩家从离线恢复（`offline=false`）后蒙层与徽章自动移除；本地玩家本人不渲染蒙层（以顶部“重连中…” Toast 提示为准），Bot 玩家从不渲染蒙层。

## 断线重连

针对网络抖动、切后台过久或小游戏被强制关闭等场景，提供「重连」/「重新进入」能力：

### 1. WebSocket 自动重连

- 客户端检测到连接断开后，自动尝试重新连接最多 5 次，间隔 1.5 秒。
- 重连过程中顶部 Toast 提示「重连中…(n/5)」。

### 2. 业务层自动恢复

- 重连成功后自动重发 `LOGIN`；若本地仍持有 `databus.room`，则在收到 `LOGIN_OK` 后自动发送 `JOIN_ROOM` 回原房间。
- 服务端命中重连分支时，会先单播一条 `RECONNECT_SNAPSHOT`：携带当前阶段、手牌、已提交的三道、最近一局的结算结果、对局进度。
- 客户端据此还原本地数据，并按阶段进入对应子界面（摆牌 / 等待开牌 / 比牌 / 整场结束）。

### 3. 大厅「重新进入」入口

- 登录成功后，若服务端检测到该 openid 仍在某个未结束房间，会在 `LOGIN_OK.data.activeRoom` 中带回房间信息。
- 大厅页主按钮上方显示「重新进入（房间号）」按钮；点击后直接回到该房间，恢复全部对局状态。
- 若房间已结束，按钮自动隐藏并提示「房间已不存在」。

### 4. 服务端 30 秒兜底

- 玩家断线后阶段化兜底（30 秒为窗口）：
  - **准备阶段**：连接断开瞬间即移除该玩家
  - **对局阶段（playing）**：标记 `offline=true`，30 秒内回来视为重连；30 秒后未回则按散牌（头 3/中 5/尾 5）参与本局结算
  - **比牌阶段（comparing）**：保留座位（结算结果与积分已固定），等待重连或整场结束
  - **整场结束（match_end）**：30 秒后未回则移除该玩家，避免永久占座
- 兜底执行后若房间内已无任何在线真人，立即销毁房间，避免房间与 bot 持续空转
- 同 openid 新建连接时，服务端会断开旧连接、把旧 Session 的房间归属继承到新 Session，并取消该玩家的弃局兜底计时器。

## 调试日志面板

为方便开发者在真机或微信开发者工具外定位卡顿、协议问题，前端新增「调试日志面板」开发期模块（默认开启，发布时一行关闭）。

### 1. 入口

- 画布右上角常驻一个浮动 `LOG` 按钮（圆角矩形 44×28，半透明）；
- 房间页位于「退出」按钮左侧 8px，避免重叠；其他页面贴右；
- 当面板关闭、缓冲区中存在新增 `ERROR` 日志时，按钮右上角显示红点提示；点击面板后清零。

### 2. 日志收集

- **Console**：替换 `console.log/info/warn/error/debug`，原始输出仍保留至开发者工具 Console；
- **HTTP**：所有 `wx.request` / `fetch` 请求记录方法、URL、请求体、状态码、耗时与响应体（自动屏蔽 `token` 字段）；
- **WebSocket**：连接事件 `connect/open/close/error/reconnect`，以及每条收发消息（type、reqId、payload 摘要）；`ERROR` 协议升级为 ERROR 级别、`RECONNECT_SNAPSHOT` 升级为 WARN 级别。
- 缓冲区上限默认 500 条，超限按头部丢弃。

### 3. 面板交互

- 顶部分类标签：`ALL / CONSOLE / HTTP / WS / ERROR`，单击即时过滤；
- 列表区按 `HH:mm:ss.sss [LEVEL] [SOURCE] 内容` 形式滚动展示，按级别/来源着色，超长内容截断；
- 上下滑动查看历史；松手后若处于贴底位置则继续自动跟随新日志，否则停留；
- 工具栏：
  - **暂停/恢复**：暂停期间缓冲区继续写入，但视图不再贴底刷新；
  - **清空**：清空全部缓冲；
  - **发包**：通过两步 `wx.showModal` 输入 `type` 与 JSON `data`，校验通过后调用 `socket.send(type, data)`；JSON 格式错误或 Socket 未连接时给出 Toast 提示并阻止发送；
  - **会话**：弹出当前 `databus.user / scene / room / socket` 关键字段快照，可一键复制；
- 单击单条日志弹出详情视图（不截断），提供「复制」按钮调用 `wx.setClipboardData` 写入剪贴板。

### 4. 关闭方式

在 [main.js](./js/main.js) 顶部将 `GameGlobal.DEBUG_LOG` 置为 `false`，即可一行关闭按钮、面板与所有 hook（不再拦截 `console`、不再写入网络日志），用于发布版本零开销。
